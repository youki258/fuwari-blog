根据阿里云百炼（DashScope）最新的官方接口文档，**Qwen3.5 系列模型（包括 qwen3.5-flash）在默认情况下是强制开启混合思考模式的**。

当模型在系统底层被设定为优先生成思考 Token 时，单纯用文字形式的 System Prompt（如 "Do not use reasoning"）很难彻底压制它，这也是为什么你的翻译依然很慢的原因。

要从根本上解决这个问题，请按照以下三种途径操作（**强烈推荐优先使用第一种**）：

### 途径一：通过 API 核心参数彻底关闭（最有效）

阿里云为 Qwen3.5 系列提供了一个专属的控制参数 `enable_thinking`。

在你的 LunaTranslator 界面最下方找到 **“其他参数”** 右侧的粉色 **`+`** 号：

1. 点击 **`+`** 号添加一个新字段。
2. 参数名（Key / 键名）填写：`enable_thinking`
3. 参数值（Value / 键值）填写：`false` （注意全部小写。如果 LunaTranslator 支持选择类型，请选择 Boolean/布尔值；如果是纯文本框，直接写 false）。

*原理解析：这会在发送给阿里云的 API 请求 JSON 中直接写入 `"enable_thinking": false`，从服务器端直接砍掉思考过程，首字响应时间（TTFT）会大幅缩短。*

### 途径二：调整为官方推荐的“非思考模式”采样参数

Qwen 团队官方文档指出，在使用“指示模式（即非思考模式）”时，如果不调整采样参数，模型可能会陷入过度重复或卡顿。请在你的截图中将参数修改为官方推荐的最佳实践：

- **Temperature**: 调高至 `0.7`（你截图里是 0.2，太低反而容易导致模型在寻找最优词时陷入死循环）。
- **top_p**: 调高至 `0.8`（你截图里是 0.3）。
- **frequency_penalty / presence_penalty**: 尝试设为 `1.5`（有助于减少模型在翻译时无休止的自我重复）。

### 途径三：System Prompt “标签截断法” (Prompt Hack)

如果由于软件兼容性导致第一步的 API 传参没生效，你可以利用推理模型底层的训练逻辑，用“伪造闭合标签”来欺骗模型，让它以为自己已经思考完了。

请在你的 System Prompt 的**最末尾**，加上以下内容（注意保留 `</think>` 这个标签）：

```text
</think>
直接输出最终的中文翻译结果，严禁输出任何分析、解释或多余的标点。
```

*原理解析：模型在收到请求时，通常会本能地想输出 `<think>` 开始思考。你在 Prompt 末尾直接喂给它一个闭合的 `</think>` 标签，可以直接阻断它的内部思维链触发机制，逼迫它直接进入正文生成阶段。*

**总结操作**：在最下方 `+` 号添加 `enable_thinking = false`，然后把 Temperature 调回 `0.7`，你的翻译速度应该就能恢复到正常的极速流式输出了。
