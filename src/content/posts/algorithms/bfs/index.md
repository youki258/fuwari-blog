## bfs 广度优先搜索

> MC0329都市路径

### BFS（广度优先搜索）算法总结

#### 1. **基本概念**
- **BFS（Breadth-First Search）** 是一种用于遍历或搜索图或树的算法。
- 从起始节点开始，逐层访问所有相邻节点，再依次访问这些节点的相邻节点，直到遍历完所有节点或找到目标节点。

#### 2. **核心思想**
- **队列（Queue）**：BFS使用队列来存储待访问的节点，确保按层级顺序访问。
- **层级遍历**：先访问离起始节点最近的节点，再逐步向外扩展。

#### 3. **算法步骤**
1. 将起始节点放入队列，并标记为已访问。
2. 从队列中取出一个节点，访问其所有未被访问的相邻节点，并将这些节点加入队列。
3. 重复步骤2，直到队列为空。

#### 4. **代码框架（伪代码）**
```python
def BFS(start):
    queue = [start]
    visited = set([start])
    
    while queue:
        node = queue.pop(0)
        for neighbor in node.neighbors:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
```

#### 5. **应用场景**
- **最短路径问题**：在无权图中，BFS可以找到从起点到目标节点的最短路径。
- **连通性检测**：判断图中两个节点是否连通。
- **层级遍历**：如二叉树的层序遍历、社交网络中的“朋友推荐”等。

#### 6. **时间复杂度与空间复杂度**
- **时间复杂度**：O(V + E)，其中V是节点数，E是边数。
- **空间复杂度**：O(V)，最坏情况下需要存储所有节点。

#### 7. **与DFS的区别**
- **BFS**：按层级遍历，适合最短路径问题；需要更多内存（队列存储）。
- **DFS**：按深度遍历，适合路径存在性问题；内存占用较少（递归栈）。

#### 8. **注意事项**
- **避免重复访问**：必须标记已访问的节点，否则可能陷入循环。
- **无权图最优性**：仅适用于无权图的最短路径，带权图需使用Dijkstra或Floyd算法。

BFS是图算法中的基础工具，核心是队列和层级遍历，适合解决最短路径、连通性等问题，是算法学习中的必备知识点。

## 前缀和，差分

### 前缀和与差分总结

#### 1. **前缀和（Prefix Sum）**

- **定义**：前缀和数组 `S`，其中 `S[i] = a[1] + a[2] + ... + a[i]`。

- 

	作用

	：

	- 快速计算区间和：`sum(l, r) = S[r] - S[l-1]`。
	- 适用于静态数组（元素不频繁修改）。

- 

	时间复杂度

	：

	- 预处理：`O(n)`。
	- 查询：`O(1)`。

- 

	应用场景

	：

	- 区间求和问题。
	- 二维前缀和（矩阵区域和）。
	- 结合哈希表优化子数组问题（如和为k的子数组数量）。

#### 2. **差分（Difference Array）**

- **定义**：差分数组 `d`，其中 `d[i] = a[i] - a[i-1]`（`d[1] = a[1]`）。

- 

	作用

	：

	- 高效处理区间增减操作：对区间 `[l, r]` 加 `k`，只需 `d[l] += k` 和 `d[r+1] -= k`。
	- 还原原数组：对差分数组求前缀和。

- 

	时间复杂度

	：

	- 区间操作：`O(1)`。
	- 还原数组：`O(n)`。

- 

	应用场景

	：

	- 频繁区间修改（如批量增减）。
	- 统计覆盖次数（如MC0351问题）。
	- 结合前缀和实现高效更新与查询。

#### 3. **对比与联系**

- 

	前缀和 vs 差分

	：

	- 前缀和用于快速查询，差分用于快速修改。
	- 差分是前缀和的逆操作（差分数组的前缀和是原数组）。

- 

	结合使用

	：

	- 差分处理区间修改后，通过前缀和还原数组。
	- 例如：先差分批量更新，再前缀和得到最终数组。

#### 4. **典型例题**

- 

	前缀和

	：

	- 一维：区间和、子数组和等于k。
	- 二维：矩阵区域和（LeetCode 304）。

- 

	差分

	：

	- 区间增减（如MC0351的覆盖统计）。
	- 航班预订统计（LeetCode 1109）。

#### 5. **代码模板**

```c++
// 前缀和
vector<int> prefix(n + 1, 0);
for (int i = 1; i <= n; ++i) 
    prefix[i] = prefix[i-1] + a[i-1];

// 差分
vector<int> diff(n + 2, 0);
void add(int l, int r, int k) {
    diff[l] += k;
    diff[r+1] -= k;
}
// 还原
for (int i = 1; i <= n; ++i) 
    a[i] = a[i-1] + diff[i];
```

- **前缀和**：查询优化，空间换时间。

- **核心思想**：通过预处理将复杂操作转化为简单运算，适合大规模数据场景。

### **字典序最小的定义与详解**

字典序（Lexicographical Order）是一种基于字母表或数字顺序的排序方式，类似于字典中单词的排列方式。在计算机科学中，字典序常用于字符串、数组或序列的比较和排序。

------

**1. 字典序最小的定义**

- **字典序最小**：在一组可能的排列中，按照从左到右逐字符（或逐元素）比较，第一个不同的位置上数值最小的排列。

- 

	比较规则

	：

	- 从第一个元素开始比较。
	- 如果相同，继续比较下一个元素。
	- 如果某个位置的元素不同，较小的那个排列字典序更小。
	- 如果一个排列是另一个的前缀，则较短的排列字典序更小。

**示例：**

- 字符串

	 

	```
	"apple"
	```

	 

	和

	 

	```
	"apricot"
	```

	：

	- 前两个字符相同 `"ap"`。
	- 第三个字符 `'p' < 'r'`，所以 `"apple"` 字典序更小。

- 数组

	 

	```
	[1, 2, 3]
	```

	 

	和

	 

	```
	[1, 3, 2]
	```

	：

	- 第一个元素相同 `1`。
	- 第二个元素 `2 < 3`，所以 `[1, 2, 3]` 字典序更小。

## **2. 如何构造字典序最小的序列**

### **(1) 贪心算法**

- **核心思想**：每一步选择当前可用的最小元素，确保后续仍然能构成合法序列。

- 

	适用场景

	：

	- 字符串拼接（如合并多个字符串使字典序最小）。
	- 排列问题（如给定约束条件下构造最小排列）。

**示例：构造字典序最小的字符串**

- 给定字符 `{'a', 'b', 'c'}`，字典序最小的排列是 `"abc"`。
- 如果允许重复，如 `{'a', 'a', 'b'}`，最小排列是 `"aab"`。
