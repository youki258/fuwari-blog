对于你**已经打开的应用、已经建立的 TCP 连接**（比如正在刷的网页、登着的微信、打着的游戏），NAT 设备会**永久保留这个会话的映射关系**：

- 哪怕公网 IP 变了，已经建立的连接不会断，应用完全不受影响，你用着和平时一模一样。
- 只有当你发起**全新的连接**（比如新开一个 SSH、重新登录某个服务、新开浏览器标签页），才会用新的公网 IP。

### 3. 互联网应用根本不关心你的源 IP

现在的所有互联网应用（微信、浏览器、视频软件等），只需要「网络通」，完全不依赖固定的源公网 IP：

- 源 IP 变了，应用不会报错、不会掉线，你根本察觉不到任何异常。
- 只有**需要「源 IP 白名单」的服务**（比如你服务器的安全组、企业 VPN、特定后台系统），才会因为源 IP 变了而被拦截，这时候你才会发现 IP 换了 —— 这就是你之前 SSH 连不上的根本原因！

### 4. 链路切换是毫秒级的

校园网的负载均衡、故障切换都是毫秒级完成的，你根本感觉不到断网，自然不会意识到 IP 变了。
