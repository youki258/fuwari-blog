### 核心结论（彻底解决你的疑惑）

**if/else 确实是「二选一」，但这是「同一个进程内部」的二选一！

而 fork () 之后，变成了 两个独立的进程，两个进程都会执行这个 if/else 判断，各自选自己的分支！**

不是「一个进程选一条路」，

是 **两个进程，各走各的路，一个走 if，一个走 else**！

---

# 一步步拆解，你马上懂

## 第一步：fork 之前 → 只有 **1 个父进程**

c

运行

```
// 此时：只有 父进程 自己在跑
while((p1 = fork()) == -1);
```

## 第二步：执行 fork () → **1 个进程 变成 2 个独立进程**

系统复制出一个一模一样的 **子进程**，现在有：

1. **父进程**（原来的）
2. **子进程**（刚复制的）

⚠️ 关键：**两个进程完全独立，都会继续往下执行代码**

## 第三步：两个进程，拿到的 p1 值不一样

- **子进程**：`fork()` 返回 `0` → `p1 = 0`
- **父进程**：`fork()` 返回 子进程的 PID → `p1 = 一个大于0的数`

## 第四步：两个进程，**各自独立** 执行 if/else 判断

c

运行

```
if(p1 == 0) {
    分支A
} else {
    分支B
}
```

1. **子进程** 看到 `p1=0` → 进 `if` 分支
2. **父进程** 看到 `p1≠0` → 进 `else` 分支

---

# 用最通俗的比喻：复制人

你（父进程）有一张任务卡：

> 如果你的编号是 0 → 做任务 A
> 
> 否则 → 做任务 B

然后你用 **fork () 复制了一个新的你（子进程）**：

- 复制人（子进程）编号 = 0 → 去做任务 A
- 原来的你（父进程）编号 ≠0 → 去做任务 B

**两个人同时行动，各做各的，互不干扰！**

# 回到你的代码，可视化跑一遍

c

运行

```
while((p1 = fork()) == -1);

// 👇 下面这段代码，两个进程都会执行！
if(p1 == 0) {
    // 子进程：走这里！打印 b
    putchar('b');
} else {
    // 父进程：走这里！继续创建第二个子进程
    while((p2 = fork()) == -1);
    ...
}
```

### 最终结果

- **子进程** → 进 `if` → 打印 b，结束
- **父进程** → 进 `else` → 创建第二个子进程

# 最终回答你的问题

### Q：if/else 不是二选一吗？为什么父进程能进 else？

### A：

1. **if/else 是单个进程内部的二选一**；
2. **fork 后变成了两个独立进程**；
3. **两个进程分别执行判断**：
    
    - 子进程 → 满足 `p1==0` → 进 `if`
    - 父进程 → 不满足 → 进 `else`
    
4. **两个分支都被执行了，只是由不同的进程执行！**
